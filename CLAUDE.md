# myHealth — Claude Guidelines

## Design Context

### Users
A single private user (the developer themselves) logging daily mood, pain levels, and habits. The context is intimate and personal — this is a daily ritual, not a productivity dashboard. There is no audience, no sharing, no performance. Every interaction should feel like writing in a private journal.

### Brand Personality
Intimate, gentle, atmospheric. The app is a quiet companion for difficult days. It holds data without judgment. Think: a well-designed notebook in a dark room, not a medical portal.

**Three words:** private · warm · composed

### Aesthetic Direction
- **Dark-only** — pure black background with soft rose/pink radial spotlights that feel ambient, not decorative
- **Manrope** — the humanist sans-serif reinforces warmth without being casual
- **Accent**: rose/pink (`#ff5e8a`, `#ff8fb1`) — expressive but not alarming, personal not clinical
- **Spatial rhythm**: generous breathing room, subtle borders, deep shadows — nothing feels cramped
- **Anti-references**: no enterprise dashboards, no SaaS "health" apps, no gamification, no bright wellness-app palettes

### Design Principles
1. **Calm over clever** — never introduce visual noise, animation, or UI complexity that raises cognitive load. Logging health data is often done on hard days.
2. **Private by feel** — the interface should feel like it belongs only to the user. No patterns that imply "sharing", "public", or "performance".
3. **Restraint is the aesthetic** — when in doubt, remove. The pink accent earns its place by being used sparingly. Empty space is intentional.
4. **Warmth without cheer** — the app deals with pain and depression data. Avoid upbeat microcopy, bright celebratory states, or anything that feels dismissive of difficult emotions.
